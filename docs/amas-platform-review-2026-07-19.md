**両者共通の空白**: 実行監査ログ（承認者・タイムスタンプ）、API入稿とその失敗/再実行管理、入稿後状態同期。Switch Media側は「CSVを人が運ぶ」ことで回避しており、AMASが自律化した瞬間にこの空白が顕在化します。

## 13. Structural Redesign Options

各案は選択肢の提示であり、**確定事項ではありません**（採否はGate 2）。

**案A: 現行AMASを基礎に段階的改善**
- 守れるコンセプト: 全て（UI/UX資産をそのまま活用）
- 再利用: AI生成基盤・UI・Stripe骨格
- 捨てる: なし
- 負債: 単一10,326行App.tsx・無認証API・偽装deploy・テスト0を抱えたまま実配信を接続することになる
- 難易度: 中（ただしP0を残したまま媒体接続すると危険度が跳ね上がる）
- 安全性: **低**（構造的欠陥の上に実弾を載せる）
- 実運用到達距離: 見かけ上は近いが、P0/P1を潰さない限り実配信は不可
- 推奨度: **低**（デモ継続には最速だが「広告OS化」には不適）

**案B: コンセプトと資産を残しアーキテクチャ再構築**
- 守れるコンセプト: 全て
- 再利用: AI生成プロンプト群・審査ロジック・データモデルの知見・UIデザイン
- 捨てる: 単一巨大コンポーネント・無認証API層・偽装実行パス・クライアント主導状態遷移
- 負債: 一掃できる
- 難易度: **高**（フルスクラッチ寄り）
- 安全性: 高（認証・監査・冪等・権限を最初から設計）
- 実運用到達距離: 遠いが確実
- 推奨度: 中（正論だが、非エンジニア・単独運用の体制ではコストが重い）

**案C: Switch Media側の運用自動化基盤を共通コアに抽出し、その上にAMAS固有の自律判断レイヤーを構築**
- 守れるコンセプト: 全て。かつ「自律運用」の実行レイヤーを実務検証済みの入稿基盤の上に載せられる
- 再利用: 【業務知識】審査ルール・媒体契約・権限モデル／【設計パターン】二段審査・部分成功・承認UX・NGドメイン・監査履歴／【AMASコード】AI生成基盤
- 捨てる: AMASの偽装実行パス・無認証API・Switch MediaのCSV UIコード（列マップのみ抽出）
- 負債: 両者の既知欠陥（監査ログ・API入稿・実機検証）を**新設として明示的に埋める**前提で始められる
- 難易度: 中〜高（共通コアのパッケージ化＝Switch Media側の密結合[cross-repo import]の解消が前提）
- 安全性: 高（「金が動く操作を人間に残す→段階的にPAUSED入稿→Financial」の実務境界を継承）
- 実運用到達距離: **中**（実務で使われている入稿基盤を土台にできる）
- Switch Media側との独立性: 共通コアをどちらにも依存しない独立パッケージにできれば両立。できなければ二重運用リスク
- Aisleとの将来接続: IHD仕様書Phase 5が既に「Aisleプラットフォームへの統合」を想定
- 推奨度: **高**（ただし前提条件つき・§14）

**案D（追加提示）: スコープ縮小での実運用先行**
- 案A〜Cの決定前に、**1媒体（Google Ads）・PAUSED入稿まで・人間承認必須**に絞り、実API接続とE2Eを1本だけ通して「自律の1周」を実証する最小構成。CVI定義・全媒体・自律実行は後続。到達距離が最短で、どの案が現実的かの判断材料が得られる。推奨度: 中〜高（検証目的として）。

## 14. Recommended Direction

**推奨（確定ではない・Gate 2合意対象）: 案C（Switch Media基盤を共通コアに抽出＋AMAS自律レイヤー）を本線とし、着手は案Dの最小E2E実証から入る。**

理由:
1. AMASの決定的欠落は「実行レイヤー」であり、それを実務で検証されつつあるSwitch Media入稿基盤の設計・業務知識で埋めるのが最も筋がよい。
2. ただし転用の主対象は**コードではなく業務知識と設計パターン**（§11）。「コードを共通化する」前提で進めると、Switch Media側の密結合とCSV UIの欠陥を持ち込むため、共通コアの独立パッケージ化が前提条件。
3. フル再構築（案B）は正論だが、非エンジニア・単独・生成AI活用という開発体制ではリスクが高い。案C＋案Dなら段階的に安全性を確保しながら「自律の1周」を先に実証できる。
4. どの案でも、**P0（特にP0-6のSecret平文混入・API無認証・残高書き換え）を先に塞ぐことが実配信接続の絶対条件**。これは案の選択に関わらず共通の前提。

**実装着手は行いません。** この推奨は仮説であり、Gate 2でのユーザー・GPT評価と合意を経て初めて範囲が固定されます。
